============================================================

A) Training resume failed from last.pt:
- Error: checkpoint had NaN/Inf corruption.
- Impact: standard resume=True from last.pt could not continue.

B) Recovery training hit host memory issue:
- Error: DataLoader worker MemoryError during mosaic augmentation.
- Impact: training crashed before completing with previous settings.

C) API startup import issue:
- Error: cannot import YOLO from ultralytics.
- Root cause: local repo contains ultralytics source, but Python import resolution did not consistently pick it when launching main.py.

D) Frontend showed no detections on sample image:
- Root cause 1: backend was previously falling back to generic yolov8m.pt in some states.

E) M2 confusion:

============================================================
5) WHAT THE APP IS DOING RIGHT NOW (END-TO-END FLOW)
============================================================

1. User uploads image from frontend.
2. Frontend sends multipart request to /predict.
3. Backend stores image in static/uploads with UUID name.
4. M1 model runs detection on image.
5. For each detection:
   - class_name and confidence are produced.
   - if severity model exists AND class is severity-candidate, M2 classification runs on crop.
6. Backend returns:
   - m1_base64 image
   - m2_base64 image
   - counts
   - detections[] with box/conf/class_name/severity
7. Frontend renders both output panels and detection rows.
8. If user flags wrong detection, frontend sends /feedback with image_id and boolean flag.
